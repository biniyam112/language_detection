## Project Overview

This project builds a spoken language identification system that predicts the
language being spoken in an audio recording. The final system supports nine
languages: Amharic, Arabic, Chinese, English, French, Hindi, Italian, Russian,
and Spanish.

The main model is a residual convolutional neural network (CNN) trained on
128-bin log-Mel spectrograms. A bidirectional LSTM model is also included as a
comparison model to test whether a sequence-based architecture performs better
on the same audio features.

The dataset combines Mozilla Common Voice and Google FLEURS recordings,
keeping both sources distinguishable during training to improve robustness
beyond a single recording domain.

## Problem Formulation

The task is formulated as a supervised multi-class classification problem.
Given a short speech recording, the model predicts one of nine language labels:

```text
audio recording -> log-Mel spectrogram -> neural network -> language label
```

Each audio file is loaded, resampled to 16 kHz, converted to a fixed-duration
5-second waveform, transformed into a log-Mel spectrogram, and passed into the
classifier. The model is trained with cross-entropy loss so that the predicted
probability for the correct language is maximized.

## Evaluation Strategy

The project evaluates performance in following ways:

- A held-out test split is used to compute accuracy, precision, recall, and
  F1-score for each language.
- Confusion matrices show which languages are most commonly confused.
- PCA visualizations of the penultimate-layer embeddings show how well
  the learned representations separate languages in feature space.
- Generated sample predictions test the saved models on small sets of audio
  samples from different sources.
- CNN and LSTM results are compared to assess whether convolutional or
  sequential modeling is better suited for log-Mel spectrogram classification.

## Key Findings

- The residual CNN (4.08M parameters) achieved 89.3% accuracy on the
  held-out test set, outperforming the bidirectional LSTM (693K parameters)
  at 80.5%.
- On generated FLEURS sample predictions, the CNN reached 84.4% accuracy
  versus 71.1% for the LSTM, suggesting the CNN generalizes more robustly
  to out-of-distribution audio.
- Amharic, Chinese, and Hindi were the strongest classes for both models,
  reflecting their phonological distinctiveness from the other languages.
- English was the weakest class for both models, likely due to high
  within-class acoustic variation across accents and recording conditions.
- Real microphone recordings exposed a domain mismatch gap not captured
  by held-out test accuracy, driven by background noise, microphone
  variation, and spontaneous speech conditions.

## Quick Start

Since the trained models and sample audio files are included in this repository, 
we can start from testing results using sample audio files generated by FLEURS.

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Run the saved CNN checkpoint on included sample audio
python run_sample_predictions.py --model cnn

# 3. Run the saved LSTM checkpoint on the same samples
python run_sample_predictions.py --model lstm

# Optional: regenerate the sample set if samples/predict_test/ is missing
python generate_sample.py --source fleurs --samples-per-language 5

# 4. Predict one audio file with a selected model
python src/predict.py --audio path/to/audio.wav --model cnn

# 5. For live audio testing execute notebooks/microphone_language_prediction.ipynb
```

Saved outputs are organized by model type:

```text

checkpoints/lstm/best_model.pth
results/cnn/
results/lstm/
```

## Dataset Layout

Training uses up to `10,000` Mozilla clips plus up to `5,000` FLEURS clips per
language.

Expected audio layout:

```text
data/raw/
  english/
    common_voice_clip_1.mp3
    fleurs/
      english_fleurs_00001.wav
  amharic/
    common_voice_clip_1.mp3
    fleurs/
      amharic_fleurs_00001.wav
```

The source caps are configured in `src/config.py`:

```python
MOZILLA_CLIPS_PER_LANGUAGE = 10_000
FLEURS_CLIPS_PER_LANGUAGE = 5_000
MAX_CLIPS_PER_LANGUAGE = MOZILLA_CLIPS_PER_LANGUAGE + FLEURS_CLIPS_PER_LANGUAGE
```

## Model Architecture

Default model: 5-block residual 2D CNN over 128-band Mel-spectrograms (~4.08M parameters).

```text
Residual CNN blocks: 1 -> 32 -> 64 -> 128 -> 256 -> 512
Classifier: Linear(512,256) -> Linear(256,128) -> Linear(128,9)
```

Comparison model: bidirectional LSTM over Mel-spectrogram time frames (~693K parameters),
with hidden size 128 per direction (256 total). It uses the same data loading,
preprocessing, training, evaluation, and prediction scripts as the CNN.

## Audio Preprocessing

Training and inference use the same audio duration and feature settings:

```text
Sample rate: 16 kHz
Duration: 5 seconds
Features: 128-bin Mel-spectrograms (FFT size 1024, hop length 512)
```

By default, training uses cached deterministic Mel-spectrograms for speed,
with SpecAugment-style frequency and time masking applied on-the-fly for
regularization. Set `TRAIN_CACHE_FEATURES = False` in `src/config.py` to
enable live waveform augmentation instead. Validation, test, and inference
always use deterministic preprocessing.
